Hephaestus Integration

Governed at every step

Three Hephaestus components wire into FORGE to bring production-grade robustness to autonomous work.

Protocol
Stormbreaker
99.26% operational robustness vs 76.48% native
Every worker output passes through 8 mandatory gates before advancing. scope_lock, contract, failure_memory, verifier_first_plan, evidence_loop, review_gate, outcome_ledger, final_gate. Low-risk tasks skip to final_gate; all others run the full chain.
Knowledge Layer
Local Ontology
Project docs queried before external search
Hephaestus runtime at ~/.agentlas/runtime/ indexes all project documents. SCOUT queries the local ontology first — only falling back to Exa when the answer is not found locally. Fewer API calls, faster context, no hallucinated sources.
Deployment
Multi-runtime AGENTS.md
Claude Code · Codex · Gemini · Cursor · Antigravity
architect.py generates a full AGENTS.md alongside ARCHITECTURE.yaml. Routing cards on every worker YAML tell each agent runtime exactly how to execute, what tools to use, and which gate protocol governs their outputs.

Stormbreaker Protocol

8-Gate execution path

Every task unit is governed by this state machine. High-risk tasks traverse all 8 gates; LOW risk skips directly to final_gate.

stateDiagram-v2 direction LR [*] --> scope_lock : task dispatched scope_lock --> issue_contract : boundary verified issue_contract --> failure_memory : contract set failure_memory --> verifier_first_plan : no repeat failures verifier_first_plan --> evidence_loop : plan validated evidence_loop --> review_gate : evidence gathered review_gate --> outcome_ledger : QA score ≥ 8.5 outcome_ledger --> final_gate : ledger updated final_gate --> [*] : human approved scope_lock --> final_gate : LOW risk shortcut review_gate --> verifier_first_plan : QA score < 8.5 (retry) issue_contract --> [*] : contract violation (escalate) note right of scope_lock Gate 1: stays within outcome boundary end note note right of evidence_loop Gate 5: all claims traceable end note note right of final_gate Gate 8: human approval required end note
Gate 1
scope_lock
Output stays within the defined outcome boundary. Rejects scope creep before it propagates.
Gate 2
issue_contract
Worker fulfilled its stated contract for this task unit. Missing deliverable triggers escalation.
Gate 3
failure_memory
No repeat of a previously failed pattern. Failure history is persistent across loop restarts.
Gate 4
verifier_first_plan
Independent verifier validates the execution plan before work begins. Catches bad plans early.
Gate 5
evidence_loop
All claims backed by traceable evidence. No unverified facts advance in the pipeline.
Gate 6
review_gate
QA score 8.5/10 or higher required. Claude Opus 4.7 scores with structured critique on failure.
Gate 7
outcome_ledger
Progress logged to the immutable outcome ledger. Every approved unit is auditable forever.
Gate 8
final_gate
Human approval required. Public safety scan runs here. Nothing leaves the system without sign-off.

Getting Started

How It Works

Three steps from outcome to autonomous Stormbreaker-governed execution.

1
🎯

Define Your Outcome

Type it in plain English, paste notes, or upload a brief. Any format works. FORGE converts it into a structured fable prompt using CL4R1T4S injection — no templates to fill out.

2
✍

Sign Off on Architecture

FORGE shows you the worker fleet it designed: ARCHITECTURE.yaml, AGENTS.md, and Stormbreaker routing cards for each worker. You approve or tweak. One gate before the loop starts.

3
📫

Check Email Hourly

FORGE runs 24/7 through the Stormbreaker 8-gate protocol until the outcome is met. Hourly email + live dashboard. Human Gate 2 controls everything that leaves. You never have to ask what is happening.

Agent Archetypes

Worker Types

FORGE assembles a fleet from these building blocks, customized per outcome. Each worker gets a Stormbreaker routing card in AGENTS.md.

SCOUT
Claude Sonnet 4.6
Research and discovery — finds targets, surfaces signals, scrapes data. Queries local Hephaestus ontology before falling back to Exa.
ontology-first
ENRICHER
Claude Haiku / Codex
Takes raw targets and fills in missing data — contacts, context, firmographics. Calls external APIs.
VALIDATOR
Claude Haiku
Checks data quality, deduplicates, verifies against known facts. Rejects bad records before they move forward.
WRITER
Claude Sonnet 4.6
Drafts emails, summaries, proposals, reports. Follows brand rules. Outputs held in human-gate queue.
QA_SCORER
Claude Opus 4.7
Scores every output 8.5/10 or higher before it advances. Provides structured critique. The quality gate that never sleeps.
REPORTER
Claude Haiku
Compiles hourly progress emails and updates the live GitHub Pages dashboard. Escalates blockers automatically.
