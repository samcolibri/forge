Getting Started

How It Works

Three steps from outcome to autonomous execution.

1
🎯

Define Your Outcome

Type it in plain English, paste notes, or upload a brief. Any format works. FORGE converts it into a structured fable prompt using CL4R1T4S injection — no templates to fill out.

2
✍️

Sign Off on Architecture

FORGE shows you the worker fleet it designed: which agents run, what tools they use, in what order. You approve or tweak. One gate before the loop starts — nothing more.

3
📬

Check Email Hourly

FORGE runs 24/7 until the outcome is met. You get an hourly email and a live dashboard. Human gates control anything that leaves the system. You never have to ask "what's happening?"

Agent Archetypes

Worker Types

FORGE assembles a fleet from these building blocks, customized per outcome.

SCOUT
Claude Sonnet 4.6
Research and discovery — finds targets, surfaces signals, scrapes data. Runs first in every pipeline.
ENRICHER
Claude Haiku / Codex
Takes raw targets and fills in missing data — contacts, context, firmographics. Calls external APIs.
VALIDATOR
Claude Haiku
Checks data quality, deduplicates, verifies against known facts. Rejects bad records before they move forward.
WRITER
Claude Sonnet 4.6
Drafts emails, summaries, proposals, reports. Follows brand rules. Outputs held in human-gate queue.
QA_SCORER
Claude Opus 4.7
Scores every output ≥8.5/10 before it advances. Provides structured critique. The quality gate that never sleeps.
REPORTER
Claude Haiku
Compiles hourly progress emails and updates the live GitHub Pages dashboard. Escalates blockers automatically.

Developer Interface

CLI Commands

Four commands. Works in any Claude Code session once the skill is linked.

FORGE CLI — Claude Code skill
# ─── Install (one-time, any machine) ─────────────────────── $ ln -s ~/projects/forge/skills/forge ~/.claude/skills/forge → Skill linked. Use /forge:run in any Claude Code session. # ─── Start a new FORGE project ────────────────────────────── /forge:run "book 5 meetings for Moreland school districts" → Injecting fable... Designing worker fleet... Awaiting sign-off. # ─── Check loop status ────────────────────────────────────── /forge:status → moreland-sdr | queued: 12 | pending_approval: 3 | done: 0 # ─── Approve pending outputs ──────────────────────────────── /forge:approve --project moreland-sdr → 3 outputs in queue. Review each → a(pprove) / r(eject) / s(kip) # ─── Stop loop (pause — resumable) ───────────────────────── /forge:stop --project moreland-sdr → Loop paused. State saved. Resume with /forge:run --resume
